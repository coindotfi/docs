## Overview

This document details our **quantum–resistant authentication system**. It explains:

- **Public Keys:** The only way to identify a wallet owner.
- **Private Keys:** The only way an owner can sign and send messages.
- **Quantum One–Time Pad (QOTP):** A dynamic, color–coded pad that only the owner can solve.

<hr />

## Public Keys  
*The only way to identify an **owner** of a wallet:*  
- `coin1hj5fveer5cjtn4wd6wstzugjfdxzl0xpdpsrzs`

## Private Keys  
*The only way an **owner** can sign and send messages:*  
- `123SomePr1vat3Key...`

## Quantum One–Time Pad (QOTP) Legend

Our QOTP is a **40–character sequence** that cycles through four colors. The full domain consists of:

- **A–Z** (26 characters)
- **0–9** (10 characters)
- **♦, ♣, ♥, ♠** (4 characters)

Each character’s color is determined by its zero–indexed position modulo 4:

- **Index mod 4 = 0:** <span style={{ color: "#e57373" }}>Red</span> → **UP**
- **Index mod 4 = 1:** <span style={{ color: "#81c784" }}>Green</span> → **DOWN**
- **Index mod 4 = 2:** <span style={{ color: "#64b5f6" }}>Blue</span> → **LEFT**
- **Index mod 4 = 3:** <span style={{ color: "#ffd54f" }}>Yellow</span> → **RIGHT**

## Quantum One–Time Pad  
Only the owner can authenticate using the one–time pad. For example, once the pad appears the solution to the pad key may be:

- <span style={{ color: "#ffd54f" }}>L</span><span style={{ color: "#64b5f6" }}>O</span><span style={{ color: "#81c784" }}>V</span><span style={{ color: "#e57373" }}>E</span>

This corresponds to:
- **L** (index 11): Yellow → **RIGHT**
- **O** (index 14): Blue → **LEFT**
- **V** (index 21): Green → **DOWN**
- **E** (index 4): Red → **UP**
